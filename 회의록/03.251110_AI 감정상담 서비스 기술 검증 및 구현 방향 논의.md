- **2명**: 실현 가능성(기술적 구현) 분석 및 테스트



------



### **2️⃣ 데이터 관련 논의**

- **필요 데이터**

  

  - 음성 감정 데이터셋
  - 텍스트 감정 분석용 데이터셋
  - (우선순위) 감정 분류가 가능한 **텍스트 데이터셋**

  

- **AI 학습 여부**

  

  - 직접 모델을 학습시킬 목적이라면 **음성 억양 데이터** 필요
  - 단순 API 호출(OpenAI 등) 기반이면 **학습 불필요**

  

- **기술 난이도**

  

  - 억양 분석 중심은 **구현 난이도 높음**
  - 현실적으로는 **음성을 텍스트로 변환 후 텍스트 감정 분석**이 유리

  

- **추가 의견**

  

  - **감정 피드백을 AI가 직접 제공하면 GPT와 차별성 약함**
  - → AI는 **감정 분석 및 요약(키워드 추출)** 중심으로 활용
  - 상담사 또는 외부 연계 요소로 **차별화 필요**

  

------



### **3️⃣ 기술 구현 및 구조 논의**

- **바이브코딩 + 서버리스 구조** 시도 중이나

  

  - AI 로직(모델 삽입, 분석 API 연동)에 어려움 존재

  

- **앱의 기본 형태 결정 필요**

  

  - **일기 녹음형 / 단방향 음성 입력** 중 택1
  - **홈 화면 vs 녹음 화면 우선 노출** 고민 중

  

- **부가 기능 제안**

  

  - “그날의 감정 기록”, “감정 통계” 등 추가 가능

  

- **기술 검증 핵심**

  

  - 음성 → 텍스트 변환 정확도 (리콜률 확인)
  - 상담사 연결 프로세스 구현 가능 여부

  - 기술 검증(리콜·정확도·API 응답 속도 등) 진행

### **5️⃣ 서비스 방향성 논의**

- **전화 기반 서비스로 가자는 의견 다수**

  

  - “무디”와 차별화 포인트 확보
  - 단순 대화형 AI가 아닌 **‘전화 경험’을 재현하는 AI**

  

- **콜백 구조에 대한 논의**

  

  - 리콜이 올 때 **상담사 연결인가? AI 전화인가?**
  - → AI가 **키워드 기반 요약 및 위로 피드백** 제공 방향 검토 중
